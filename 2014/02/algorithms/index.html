Quick-union

I quickly skipped past the first couple of lectures on unions to get to quick-union. This strategy works with trees of nodes and more efficiently finds if two nodes are connected by finding each node’s root and determining if they’re the same root. To connect nodes, you connect their roots. Unfortunately, it’s still a pretty slow strategy, mainly because the trees can get very tall.

Weighted quick-union

This strategy is a modification of quick-sort so that the trees don’t get too tall. Basically, when we’re connecting a tall tree with a shorter tree, we make sure to put the smaller tree as the child of the bigger tree. When we’re adding a node that’s nested in a different tree, we add that node’s entire tree as a child of the tree we’re moving it to. By adding path compression, we flatten each tree out. Not quite sure why we’re doing this; professor says it’s “because we can.”
